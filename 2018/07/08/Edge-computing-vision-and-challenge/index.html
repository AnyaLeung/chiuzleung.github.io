Edge Computering: Vision and Challenges

part1: overall

5 parts in this paper, including overall, why needs and defination, case studies, challenges and oppotunities, together with conclusion.

Mainly compared with cloud computing.

part2: why needs and defination

why needs

1.Cloud’s push: Cloud computing isn’t good enough

a Latency is too long and too much data are transferred to cloud.

b Privacy.

c 无线传输费电

2.IoT’s pull: Electrical devices ⬆️⬆️⬆️

Electrical devices turn from data consumer to data consumer + productor. 未来很多它们的数据都会在edge被处理，而非传到cloud.

3.Data consumer -> data consumer + productor

defination

Edge computing: computing on the edge of the network. Computing happens near the proximity of data sources.


Edge可以是一个micro data centre.

benefits

Fast. Save energy.

part3: case studies

cloud offloads

Cloud: centralized, difficult to upload to cloud and download from cloud.

Edge: has certain computing capability, cloud can offload to edge and computing faster.

z.b. 导航软件把邻近地区的地图放在附近的edge上；信息筛选/aggregating在edge上做，⬇️transfer信息量；real-time app, ⬇️latency

part4: challenges and oppotunities

programmability

Cloud: user deploy application to cloud, cloud provider decide where to compute in cloud, with all these unknown to user

Edge: from cloud offloads to edges, edge nodes may in different platforms, making it difficult for users to write application to deploy in => computing stream

Computing stream: a serial of fuctions/computing during data propagation path, functions/computing can be part/whole of an application, application defines where computing should be conducted.
In computing stream, functions可以复用, stream要带着修改过的data, state, collebotive信息

naming

都不够成熟的NDN, Mobile First. 我们要开发一个新的！

z.b. smart home:



This blocks hardware info from user.

data abstraction

  • Different things 的data report的format不一样 => define as


  • Data abstraction 的degree不好确定，太少了不够app用，太多了size太大，有时候还不可靠
  • Applicable operations on the things. data abstraction layer is an public interface serving all things connected to edgeOS.

service management

  • 1.Differentiation: 不同的service有不同的优先级

    2.Extensibility: 高扩展性。Things in IoT are very dynamic, like insuring there’s no problem while adding a new thing.
  • 3.Isolation:

    用同样data src 的app们，有一个崩溃了，不能让别的也崩溃。/当one thing从IoT中撤出时，无影响。 => deployment/undeployment framework

    隔绝user的data和第三方application => control access mechanism
  • 4.Reliability:

    a. service: 通知user哪个service没有响应，或是在系统可能崩溃时发出警报

    b. system: 保证整个system的网路稳定

    c. data: transmission protocol is not reliable enough
  • 5.privacy and security:

    a. 人们缺少⬆️意识, z.b.随便连WiFi

    b. edge端收集的data的ownership => fully by user

    c. tool to protect data on edge

optimization metrics(指标

  • 1.latency: computation time + WAN delay -> latency, nearst layer with enough computation capability
  • 2.bandwidth: high bandwidth reduces transmission time, establish high bandwidth wireless from data src to edge.

  • 3.energy and cost:
    决定是local computing / transmission to edge computing: local computing energy & transmission energy

    total energy = sum of each layer's energy
each layer's energy = local computation cost + transmission cost
