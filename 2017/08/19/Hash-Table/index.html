Hash Table

一种数据结构，由数组(value) 和Hash Function组成，输入由Hash Function映射到数组索引，相同的输入得到相同的索引。

适用范围

  • 模拟映射关系
  • 大海捞针的寻找，如DNS解析（域名->IP地址)
  • 防止重复
  • cookie，原理：让服务器把网页记住，需要的时候直接发给用户


实现

各种语言都提供了散列表的视线，python中是字典，如：


collision(冲突)

两个键映射到同一个位置

solution: 在这个位置存储一个链表


时间复杂度

  • 平均情况下，查找、插入、删除的时间复杂度都是O(1)

    而O(1)不意味着马上，而是不管规模多大，所需时间相同 。因为我们是通过给Hash Function传入key而获得value的，假设这个位置上只有一个值，在平均情况下时间复杂度是O(1)，而且不随规模增大而改变

  • 最糟情况(collision)存在大量冲突，时间复杂度是O(n)

    冲突时的时间复杂度是O(1) + O(load_factor) = O(1 + load_factor)，所以最糟情况就是O(n)


性能

Hash Table避免冲突有赖于较低的填装因子和良好的散列函数

装填因子

调整长度(resizing)：当装填因子太大的时候，需要在散列表中添加位置

装填因子越低，效率越高

当装填因子大于0.7，需要调整长度。调整长度的开销很大，但平均而言，考虑调整长度的开销，时间复杂度也是O(1)(喵喵?)

散列函数

良好的散列函数让数组中的值均匀分布，而糟糕的导致大量冲突(可以看一下SHA函数)


总结

  • 哈希表一般不用自己实现，编程语言提供了它的视线，并假定时间复杂度是O(1)。
  • 自己实现的话，可以用数组和Hash Function。
  • Hash Function应尽量避免冲突，并降低装填因子。
  • 装填因子大于0.7时，需要resizing。
  • 哈希表用于模拟映射关系、防重、存储cookie。
